# UI Polish & Stability Pass — 2026-07-23

Approved scope (user pre-approved everything, no check-ins needed): stability fixes,
color/symbology discipline, globe entry fixes, panel content upgrades, cinematics.
Source: browser-driven UI review of the live console.

## Execution model

Five parallel lanes with **strict file ownership**. No lane edits a file owned by
another lane. `assets/css/console.css` and `index.html` are owned by the
**integrator** (main session): lanes report needed CSS/HTML in their final report
as a fenced block titled `CSS ADDITIONS` / `HTML CHANGES`, with exact selectors.
Lanes may create DOM nodes from JS (existing pattern: `control.js` manual banner).

Style rules for all lanes: vanilla ES2019 IIFE modules matching existing files, no
build step, must work from `file://`, comments only for non-obvious constraints,
uppercase console voice ("SOVEREIGN, PRECISE"), red `#ff5a5a`/`#BC0000` reserved
for brand + alerts only. Run `node --test tests/*.test.js` before reporting done.

## Cross-lane contracts

- C-1 `engine.createMission` (Lane D) throws `WAYPOINT OUTSIDE COVERAGE` when any
  waypoint exceeds `1.05 ×` the dock's range (`DOCK_RANGE.dockRangeKm`).
- C-2 Engine (Lane D) adds `code:'MISSION_LAUNCHED'` to the launch emit in
  `createMission`, and emits level `'alert'` (not `'warn'`) for forced RTB and
  dock fault events, so UI code never regex-matches message copy.
- C-3 Map (Lane B) exposes `EC2.launchPulse(dockId)` — expanding ring FX at the
  dock. Main loop (Lane A) wires `engine.onEvent` → `code==='MISSION_LAUNCHED'` →
  `EC2.launchPulse(ev.dockId || derive from source)`. Engine includes `dockId` on
  that event's object (Lane D).
- C-4 Map (Lane B) exposes `EC2.setRangeHighlight(dockId|null)` — bright coverage
  ring + fill for one dock. Control (Lane D) calls it entering/leaving wizard
  step 2 and manual mode. Must be safe to call before map load (no-op).
- C-5 Map (Lane B) reads `EC2.state.selection` + `EC2.followDroneId` to decide
  the spotlit mission route; no new API needed. Selection-change re-render:
  panels already call `EC2.updateLiveLayers` each frame; Lane B must include
  selection in its missions cache key.
- C-6 Lane A simplifies `EC2.eventLevel` to trust `ev.level` (depends C-2; write
  it level-trusting from the start — Lane D lands the engine side in parallel).

  type (mission type key), place (uppercase area name), coords, priority
  'ROUTINE'|'PRIORITY'|'URGENT', params:{altM,speedMs}, requestedAt (sim s),

  'declined', info emit. finalizeMission marks the linked request 'completed'.
